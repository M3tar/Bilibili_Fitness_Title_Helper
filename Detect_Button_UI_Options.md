# 识别与填写按钮 UI 位置方案

## 背景

后续会增加一个更强的动作：

```text
一键填写到 B 站
```

它可能会自动填写标题、选择创作声明、分区等内容。这个动作比“复制标题”风险更高、影响更大，所以 UI 上需要明确区分三类动作：

| 动作 | 性质 | 建议权重 |
| --- | --- | --- |
| 识别页面 | 读取当前投稿页，刷新预览 | 辅助按钮 |
| 复制标题 | 复制当前预览文本 | 次主按钮 |
| 填写到 B 站 | 写入投稿页表单并选择配置 | 主按钮，但需要确认感 |

当前采用轻量分步工作流。核心原则：

- “识别当前投稿页”是高频第一动作，放在顶部整行展示。
- “填写到 B 站”靠近最终操作区，因为它是提交前的最后一步。
- “复制标题”保留，但权重低于“填写到 B 站”。

## 当前采用：轻量分步工作流

```text
┌──────────────────────────────┐
│ B站健身视频标题生成器          │
├──────────────────────────────┤
│ [识别当前投稿页]  ← 顶部醒目主入口 │
│ 识别到日期：2026-06-05 周五   │
│ 疑似博主：张峰                │
│ 周五类型：手臂核心            │
│                              │
│ 选择日期                      │
│ [2026/06/05 周五] [今天][昨天] │
│                              │
│ 选择博主                      │
│ [祖嘉泽] [张峰]               │
│                              │
│ 时间段 / 类型                  │
│ [早间] [晚间]                 │
│                              │
│ 标题预览                      │
│ ┌──────────────────────────┐ │
│ │「2026/06/05 周五｜晚间」...│ │
│ └──────────────────────────┘ │
│                              │
│ [复制标题]                    │
└──────────────────────────────┘
```

优势：

- 高频第一动作足够显眼，打开插件不用找。
- 未识别前摘要不占位，按钮和日期区域的距离更紧凑。
- 摘要紧跟识别按钮，反馈直接。
- 后面是人工确认和修改区域，顺序符合“读取 -> 校正 -> 输出”。
- 后续加“填写到 B 站”时，可以放到底部，与“复制标题”组成输出区。

后续底部可以升级为：

```text
[复制标题]              [填写到 B 站]
```

## 方案 A：预览区识别 + 底部双操作

```text
┌──────────────────────────────┐
│ B站健身视频标题生成器          │
├──────────────────────────────┤
│ 选择日期                      │
│ [2026/06/05 周五] [今天][昨天] │
│                              │
│ 选择博主                      │
│ [祖嘉泽] [张峰]               │
│                              │
│ 时间段                        │
│ [早间] [晚间]                 │
│                              │
│ 标题预览              [识别页面]│
│ ┌──────────────────────────┐ │
│ │「2026/06/05 周五｜晚间」...│ │
│ └──────────────────────────┘ │
│ 识别到日期：2026-06-05 周五   │
│ 疑似博主：张峰                │
│ 周五类型：手臂核心            │
│                              │
│ [复制标题] [填写到 B 站]       │
└──────────────────────────────┘
```

优势：

- 识别按钮和标题预览强关联，用户知道点击后会影响预览。
- “填写到 B 站”在底部最终操作区，符合确认完成后的动作顺序。
- 可以把“填写到 B 站”做成主按钮，“复制标题”做成次按钮。
- 适合后续增加创作声明、分区等自动填写能力。

缺点：

- 底部两个按钮会比单个“复制标题”占更多横向空间。
- 需要设计好按钮文案，避免用户误点“填写到 B 站”。

建议：

```text
[复制标题]        次按钮
[填写到 B 站]     主按钮
```

如果担心误填，可以把主按钮文案写成：

```text
填写到投稿页
```

或者点击后先显示一次确认摘要。

## 方案 B：识别与填写都放底部

```text
┌──────────────────────────────┐
│ B站健身视频标题生成器          │
├──────────────────────────────┤
│ 选择日期                      │
│ [2026/06/05 周五] [今天][昨天] │
│                              │
│ 选择博主                      │
│ [祖嘉泽] [张峰]               │
│                              │
│ 标题预览                      │
│ ┌──────────────────────────┐ │
│ │「2026/06/05 周五｜晚间」...│ │
│ └──────────────────────────┘ │
│ 识别到日期：2026-06-05 周五   │
│ 疑似博主：张峰                │
│                              │
│ [识别页面]                    │
│ [复制标题] [填写到 B 站]       │
└──────────────────────────────┘
```

优势：

- 所有动作集中在底部，操作区清晰。
- 用户不会在表单中间看到多个按钮，视觉更干净。

缺点：

- “识别页面”离日期和预览都远，语义弱一点。
- 如果弹窗高度紧张，底部三个动作容易拥挤。

适用场景：

- 后续如果要把底部固定成操作栏，这个方案比较稳定。

## 方案 C：页面自动化区

把“识别页面”和“填写到 B 站”组成一个独立的“投稿页操作”区。

```text
┌──────────────────────────────┐
│ B站健身视频标题生成器          │
├──────────────────────────────┤
│ 选择日期                      │
│ [2026/06/05 周五] [今天][昨天] │
│                              │
│ 选择博主                      │
│ [祖嘉泽] [张峰]               │
│                              │
│ 投稿页操作                    │
│ [识别页面] [填写到 B 站]       │
│ 识别到日期：2026-06-05 周五   │
│ 疑似博主：张峰 · 周五类型：手臂核心 │
│                              │
│ 标题预览                      │
│ ┌──────────────────────────┐ │
│ │「2026/06/05 周五｜手臂核心」│ │
│ └──────────────────────────┘ │
│                              │
│ [复制标题]                    │
└──────────────────────────────┘
```

优势：

- “读页面”和“写页面”被明确放在同一组，逻辑完整。
- 用户容易理解这两个按钮都和 B 站投稿页有关。

缺点：

- 页面上会多一个分组，弹窗更长。
- “填写到 B 站”离标题预览略远，用户可能还没确认标题就点填写。

适用场景：

- 如果后续自动选择创作声明、分区、标签等动作很多，可以考虑这个独立分组。

## 方案 D：分步工作流

用 1、2、3 的流程感组织界面。

```text
┌──────────────────────────────┐
│ B站健身视频标题生成器          │
├──────────────────────────────┤
│ 1. 读取投稿页                  │
│ [识别页面]                    │
│ 识别到日期：2026-06-05 周五   │
│                              │
│ 2. 确认标题                    │
│ [日期] [博主] [时间/类型]      │
│ ┌──────────────────────────┐ │
│ │标题预览，可手动修改          │ │
│ └──────────────────────────┘ │
│                              │
│ 3. 输出                       │
│ [复制标题] [填写到 B 站]       │
└──────────────────────────────┘
```

优势：

- 最适合复杂自动化，用户知道每一步发生什么。
- “填写到 B 站”不会显得突兀。

缺点：

- 对当前小工具来说偏重，界面会从工具面板变成流程向导。
- 高频使用时会显得啰嗦。

适用场景：

- 如果未来不仅填标题，还要自动选分区、声明、标签、封面，且失败状态较多，可以再升级成这个形态。

## 推荐方案

如果未来觉得顶部按钮太占位置，可以回到 **方案 A：预览区识别 + 底部双操作**。

最终结构建议：

```text
标题预览                         [识别页面]
┌────────────────────────────────────┐
│ 可编辑标题                           │
└────────────────────────────────────┘
识别到日期：2026-06-05 周五
疑似博主：张峰
周五类型：手臂核心

[复制标题]              [填写到 B 站（后续）]
```

理由：

- “识别页面”负责生成/刷新预览，所以放在预览标题栏右侧最自然。
- “填写到 B 站”是最终动作，放在底部更像确认后的执行。
- 未来增加“自动选择创作声明、分区”等能力时，不需要再移动识别按钮，只需要扩展底部主按钮能力。
- 保留“复制标题”作为备用路径，万一自动填写失败，用户仍然能复制手动粘贴。

按钮权重建议：

```text
识别当前投稿页：顶部蓝色整行按钮
复制标题：浅色或次级按钮
填写到 B 站：蓝色主按钮
```

安全建议：

- “填写到 B 站”第一次上线时，建议只填标题，不立刻选择声明/分区。
- 等标题写入稳定后，再逐步增加创作声明、分区、标签等自动化。
- 如果一次点击要改多个字段，建议显示填写结果摘要，例如：

```text
已填写标题
已选择分区：健身
已选择创作声明：转载/自制
```
