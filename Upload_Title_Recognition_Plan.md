# B 站投稿页识别与填写方案

## 当前状态

当前版本采用“手动识别 + 用户确认 + 一键填写”的半自动流程：

1. 用户在 B 站投稿页上传视频。
2. 用户打开插件，点击“识别当前投稿页”。
3. 插件从投稿页候选文本中提取日期，并根据关键词推断博主。
4. 插件更新日期、博主、时间段或类型，以及标题预览。
5. 用户确认或手动调整标题预览。
6. 用户点击“填写到 B 站”。
7. 插件写入标题、创作声明、分区、标签，并按博主尝试上传内置封面。
8. 用户在 B 站页面最终检查并手动发布。

插件不会点击 B 站最终发布按钮。

## 固定投稿配置

| 字段 | 默认值 |
| --- | --- |
| 创作声明 | 内容无需标注 |
| 分区 | 健身 |
| 标签 | 减肥、健身、训练 |

标签处理规则：

- 删除默认标签以外的干扰标签。
- 保留已存在的默认标签。
- 补齐缺失的默认标签。

封面映射规则：

| 博主 | 封面资源 |
| --- | --- |
| 祖嘉泽 | `covers/zu-jiaze.jpg` |
| 陈玉轩 | `covers/chen-yuxuan.jpg` |
| 张峰 | `covers/zhang-feng.png` |

## 支持的输入格式

优先支持当前稳定出现的视频标题或文件名格式：

```text
2026-06-04 19-29-00 跟练七天看变化！.mp4
2026-06-03 19-27-00 夜猫子专属!睡前减脂增肌操，快来练练吧!
```

日期提取规则：

```js
/(\d{4})-(\d{2})-(\d{2})/
```

当前暂不识别 `20260604`、`6月4日`、`06.04` 等扩展日期格式。

## 博主推断规则

| 关键词 | 推断博主 |
| --- | --- |
| 祖嘉泽 | 祖嘉泽 |
| 跟练七天看变化 | 祖嘉泽 |
| 跟练七天 | 祖嘉泽 |
| 七天看变化 | 祖嘉泽 |
| 张峰 | 张峰 |
| 夜猫子专属 | 张峰 |
| 减脂增肌操 | 张峰 |

识别策略：

- 候选文本会优先选择“日期 + 博主关键词”同时存在的内容。
- 如果最佳日期候选没有博主关键词，会从其他候选文本中兜底寻找博主。
- 如果仍然没有命中关键词，只更新日期，并保留当前已选择博主。

## 用户交互

顶部高频入口：

```text
识别当前投稿页
```

识别成功后显示摘要：

```text
识别到日期：2026-06-05 周五
疑似博主：祖嘉泽
时间段：晚间
```

识别状态：

| 状态 | 含义 |
| --- | --- |
| 未识别 | 当前内容来自默认日期 |
| 已识别 | 当前内容来自投稿页识别结果 |
| 手动 | 用户手动调整了日期、博主、时间、类型或标题 |

底部操作区：

```text
提醒：本次使用识别结果填写，请确认投稿页内容
[复制标题] [填写到 B 站]
```

填写到 B 站时，popup 会在底部操作区按步骤显示明细，提醒文案始终保留：

```text
提醒：本次使用识别结果填写，请确认投稿页内容
----------------
已填写标题
已选择创作声明：内容无需标注
已选择分区：健身
已设置标签：减肥、健身、训练
已上传封面，请检查是否需要手动确认裁剪
```

填写结果右上角提供 `×`，用于收起本次结果展示，不影响已经写入 B 站页面的内容。

底部操作区采用吸附设计：

- 未滚动到底部时，操作区固定在弹窗底部并显示大面积阴影。
- 滚动到底部完整展示时，阴影消失。
- 操作区背景使用浅色横向渐变，右侧略带淡蓝色。
- “显示陈玉轩”开关放在“选择博主”标题右侧，用于控制陈玉轩按钮是否显示，不代表直接选择陈玉轩。

## 技术方案

### manifest.json

当前需要的权限：

```json
"permissions": ["clipboardWrite", "tabs"],
"host_permissions": ["https://member.bilibili.com/*"]
```

投稿页注入：

```json
"content_scripts": [
  {
    "matches": ["https://member.bilibili.com/platform/upload/*"],
    "js": ["content.js"],
    "run_at": "document_idle"
  }
]
```

封面资源声明：

```json
"web_accessible_resources": [
  {
    "resources": ["covers/*.jpg", "covers/*.png"],
    "matches": ["https://member.bilibili.com/*"]
  }
]
```

### content.js

主要职责：

- 读取 B 站投稿页候选文本。
- 写入标题输入框。
- 选择创作声明和分区。
- 清理并设置标签。
- 打开封面设置，尝试走本地上传封面流程。
- 向 popup 发送分步填写进度。

消息类型：

```js
GET_UPLOAD_TITLE_CANDIDATES
FILL_UPLOAD_FORM
FILL_UPLOAD_PROGRESS
```

填写请求示例：

```js
{
  type: 'FILL_UPLOAD_FORM',
  payload: {
    runId: '...',
    title: '「2026/06/05 周五｜晚间」祖嘉泽有氧健身 直播回放录屏完整版',
    declaration: '内容无需标注',
    category: '健身',
    tags: ['减肥', '健身', '训练'],
    cover: {
      path: 'covers/zu-jiaze.jpg',
      filename: 'zu-jiaze.jpg'
    }
  }
}
```

逐项结果示例：

```js
{
  success: true,
  results: [
    { field: 'title', success: true, message: '已填写标题' },
    { field: 'declaration', success: true, message: '已选择创作声明：内容无需标注' },
    { field: 'category', success: true, message: '已选择分区：健身' },
    { field: 'tags', success: true, message: '已设置标签：减肥、健身、训练' },
    { field: 'cover', success: true, message: '已上传封面，请检查是否需要手动确认裁剪' }
  ]
}
```

### popup.js

主要职责：

- 管理日期、博主、时间段、类型和标题预览。
- 向当前 B 站投稿页发送识别和填写请求。
- 根据 `runId` 接收当前填写任务的分步进度，避免旧任务消息串入。
- 展示识别摘要、来源提示、填写进度和逐项结果。
- 根据滚动位置切换底部操作区的吸附阴影。
- 支持收起填写结果展示。

核心函数：

- `detectCurrentPageTitle()`
- `extractDateFromText(text)`
- `inferTrainerFromText(text)`
- `getBestDetectedResult(candidates)`
- `fillCurrentUploadPage()`
- `requestFillUploadForm(tabId, payload, callback)`
- `handleFillProgressMessage(message)`
- `updateActionStepAffixState()`
- `clearFillStatus()`

## 选择器与稳定性策略

B 站投稿页可能频繁调整 DOM，当前实现避免只依赖单一 CSS class。

字段定位优先级：

1. 通过 label 文案定位字段，例如 `标题`、`创作声明`、`分区`、`标签`。
2. 在 label 附近查找 input、textarea、combobox、dropdown。
3. 通过 placeholder、maxlength、当前值和上下文文本辅助判断。
4. 最后使用具体 class 或可见文本作为兜底。

封面定位策略：

- 优先寻找图片类 `input[type="file"]`。
- 如果没有图片上传 input，尝试点击 `封面设置` 或 `.cover-img`、`.cover-item`、`.cover-empty`、`.cover-main` 等封面区域。
- 弹窗打开后优先寻找 `本地上传`、`上传图片`、`上传封面`、`选择图片`、`自定义封面`。
- 封面流程不依赖“智能封面生成中...xx%”是否完成，只关注手动上传入口。

## 安全策略

- 不点击 B 站最终发布按钮。
- 写入前使用插件标题预览框当前文本，尊重用户手动修改。
- 标题为空或超过 80 字时阻止填写。
- 找不到某个字段时，该字段返回失败提示，不阻塞其他字段继续尝试。
- 自动封面属于弱确认：插件能确认已尝试提交封面文件，但 B 站弹窗关闭、裁剪确认和服务器保存状态仍需用户最终检查。

## 验收标准

### 识别成功路径

输入：

```text
2026-06-04 19-29-00 跟练七天看变化！.mp4
```

点击“识别当前投稿页”后：

- 日期变为 `2026/06/04 周四` 或对应博主格式。
- 博主自动切换为 `祖嘉泽`。
- 标题预览刷新为祖嘉泽标题。

输入：

```text
2026-06-03 19-27-00 夜猫子专属!睡前减脂增肌操，快来练练吧!
```

点击“识别当前投稿页”后：

- 日期变为 `2026-06-03 周三`。
- 博主自动切换为 `张峰`。
- 标题预览刷新为张峰标题。

### 填写成功路径

点击“填写到 B 站”后：

- 标题写入成功。
- 创作声明选择为 `内容无需标注`。
- 分区选择为 `健身`。
- 标签最终包含 `减肥`、`健身`、`训练`，并清理默认列表以外的干扰标签。
- 根据当前博主尝试上传对应封面。
- popup 显示分步进度和最终逐项结果，结果可收起。
- 不点击发布按钮。

### 降级路径

如果识别到日期但没有识别到博主：

- 日期更新。
- 当前已选博主保持不变。
- 标题预览刷新。

如果没有识别到日期：

- 不改变当前选择。
- 显示失败提示。

如果封面入口定位失败：

- 标题、创作声明、分区和标签结果仍正常展示。
- 封面结果显示失败提示和诊断信息。
- 用户可以手动上传封面。
