# B 站投稿页标题识别功能计划

## 当前状态

第一版已实现。当前版本采用“手动点击识别页面”的半自动方案：识别成功后只更新插件内的日期、博主和标题预览，不自动写入 B 站投稿标题框。实现上使用 `content_scripts` 仅在 B 站投稿页准备候选文本读取能力。

## 目标

第一版实现“手动识别当前 B 站投稿页”的半自动流程：

1. 用户在 B 站投稿页上传视频。
2. 用户打开插件，点击“识别当前页面”。
3. 插件读取当前投稿页中的默认视频标题或文件名。
4. 插件从标题开头提取日期。
5. 插件根据高置信关键词尝试推断博主。
6. 插件更新 popup 中的日期、博主和标题预览。
7. 用户确认预览后，点击“复制标题”。

第一版不自动写入 B 站标题框，避免误填。

## 支持的输入格式

第一版优先支持当前稳定出现的标题格式：

```text
2026-06-04 19-29-00 跟练七天看变化！.mp4
2026-06-03 19-27-00 夜猫子专属!睡前减脂增肌操，快来练练吧!
```

日期提取规则：

```js
/(\d{4})-(\d{2})-(\d{2})/
```

识别结果示例：

```text
2026-06-04
2026-06-03
```

## 博主推断规则

第一版只做高置信关键词匹配：

| 关键词 | 推断博主 |
| --- | --- |
| 跟练七天看变化 | 祖嘉泽 |
| 夜猫子专属 | 张峰 |
| 减脂增肌操 | 张峰 |

如果没有命中关键词：

- 只更新日期。
- 不改变当前已选择的博主。

## 用户交互

在 popup 日期区域附近新增一个辅助按钮：

```text
识别当前页面
```

点击后的反馈状态：

- 识别成功：显示识别摘要，包括日期、疑似博主、时间段或训练类型。
- 当前页面不可识别：显示“未在当前页面找到可识别标题”。
- 当前页面不是 B 站投稿页：显示“请在 B 站投稿页使用”。
- 注入或读取失败：显示“读取当前页面失败”。

识别成功后：

- 日期输入框更新为识别日期。
- 今天/昨天快捷按钮选中态同步更新。
- 如果推断出博主，则对应博主按钮高亮。
- 标题预览自动刷新。

## 技术方案

### 1. manifest.json

增加读取当前标签页和 B 站投稿页内容所需权限：

```json
"permissions": ["clipboardWrite", "tabs"],
"host_permissions": ["https://member.bilibili.com/*"]
```

通过 `content_scripts` 将脚本限制在 B 站投稿页：

```json
"content_scripts": [
  {
    "matches": ["https://member.bilibili.com/platform/upload/*"],
    "js": ["content.js"],
    "run_at": "document_idle"
  }
]
```

### 2. content.js

新增 `content.js`，负责在当前页面中读取候选文本。

第一版读取范围：

- `input`
- `textarea`
- 常见可见文本节点
- 页面中可能承载文件名或标题的元素文本

返回内容：

```js
{
  success: true,
  candidates: ["2026-06-04 19-29-00 跟练七天看变化！.mp4"]
}
```

如果没找到候选文本：

```js
{
  success: false,
  message: "未找到候选文本"
}
```

### 3. popup.html

新增识别按钮，建议放在日期控件附近：

```html
<button id="detectCurrentPage">识别当前页面</button>
```

如果高度紧张，可以让它与日期快捷按钮同排，或做成较弱的 secondary button。

### 4. popup.js

新增核心函数：

- `detectCurrentPageTitle()`
- `extractDateFromText(text)`
- `inferTrainerFromText(text)`
- `applyDetectedResult(result)`
- `showStatus(message)`

流程：

```text
点击“识别当前页面”
-> 获取当前 active tab
-> 检查 URL 是否为 B 站投稿页
-> 向投稿页 content.js 发送消息
-> 获取候选文本
-> 提取日期
-> 推断博主
-> 更新 selectedDate / selectedTrainer
-> renderTrainerButtons()
-> bindTrainerButtonEvents()
-> refreshControls()
-> 显示状态反馈
```

### 5. popup.css

新增识别按钮和状态提示样式：

- 识别按钮使用 secondary 样式，避免抢主操作“复制标题”的视觉权重。
- 状态提示使用小号文本，成功/失败用颜色区分。

## 非目标

第一版暂不实现：

- 自动监听 B 站投稿页变化。
- 自动写入 B 站标题输入框。
- 自动上传封面。
- 识别 `20260604`、`6月4日`、`06.04` 等扩展日期格式。
- 复杂博主判断或模糊匹配。

## 验收标准

### 成功路径

在 B 站投稿页中存在以下文本时：

```text
2026-06-04 19-29-00 跟练七天看变化！.mp4
```

点击“识别当前页面”后：

- 日期变为 `2026/06/04 周四` 或对应博主格式。
- 博主自动切换为 `祖嘉泽`。
- 标题预览刷新为祖嘉泽标题。
- 用户可以点击“复制标题”复制预览文本。

在 B 站投稿页中存在以下文本时：

```text
2026-06-03 19-27-00 夜猫子专属!睡前减脂增肌操，快来练练吧!
```

点击“识别当前页面”后：

- 日期变为 `2026-06-03 周三`。
- 博主自动切换为 `张峰`。
- 标题预览刷新为张峰标题。

### 降级路径

如果识别到日期但没有识别到博主：

- 日期更新。
- 当前已选博主保持不变。
- 标题预览刷新。

如果没有识别到日期：

- 不改变当前选择。
- 显示失败提示。
