# B 站投稿页标题识别功能计划

## 当前状态

第一版已实现。当前版本采用“手动点击识别当前投稿页”的半自动方案：识别成功后只更新插件内的日期、博主和标题预览，不自动写入 B 站投稿标题框。实现上使用 `content_scripts` 仅在 B 站投稿页准备候选文本读取能力。

第二阶段 v1 已实现：用户确认标题预览无误后，可以点击“填写到 B 站”，将当前标题预览写入 B 站投稿页标题输入框。

第二阶段 v2 已实现：在标题写入后，继续设置创作声明、分区和标签。该功能仍然不会点击 B 站最终发布按钮。

当前固定投稿配置：

| 字段 | 默认值 |
| --- | --- |
| 创作声明 | 内容无需标注 |
| 分区 | 健身 |
| 标签 | 减肥、健身、训练 |

标签处理规则：

- 删除默认标签以外的干扰标签。
- 保留已存在的默认标签。
- 补齐缺失的默认标签。

## 目标

第一版实现“手动识别当前 B 站投稿页”的半自动流程：

1. 用户在 B 站投稿页上传视频。
2. 用户打开插件，点击“识别当前投稿页”。
3. 插件读取当前投稿页中的默认视频标题或文件名。
4. 插件从标题开头提取日期。
5. 插件根据高置信关键词尝试推断博主。
6. 插件更新 popup 中的日期、博主和标题预览。
7. 用户确认预览后，可点击“复制标题”，或点击“填写到 B 站”写入投稿页。

第一版不自动写入 B 站标题框，避免误填；第二阶段已增加用户主动触发的“填写到 B 站”能力。

## 支持的输入格式

第一版优先支持当前稳定出现的标题格式：

```text
2026-06-04 19-29-00 跟练七天看变化！.mp4
2026-06-03 19-27-00 夜猫子专属!睡前减脂增肌操，快来练练吧!
```

日期提取规则：

```js
/(\d{4})-(\d{2})-(\d{2})/
```

识别结果示例：

```text
2026-06-04
2026-06-03
```

## 博主推断规则

第一版只做高置信关键词匹配：

| 关键词 | 推断博主 |
| --- | --- |
| 跟练七天看变化 | 祖嘉泽 |
| 夜猫子专属 | 张峰 |
| 减脂增肌操 | 张峰 |

如果没有命中关键词：

- 只更新日期。
- 不改变当前已选择的博主。

## 用户交互

在 popup 顶部新增一个高频入口按钮：

```text
识别当前投稿页
```

点击后的反馈状态：

- 识别成功：显示识别摘要，包括日期、疑似博主、时间段或训练类型。
- 当前页面不可识别：显示“未在当前页面找到可识别标题”。
- 当前页面不是 B 站投稿页：显示“请在 B 站投稿页使用”。
- 注入或读取失败：显示“读取当前页面失败”。

识别成功后：

- 日期输入框更新为识别日期。
- 今天/昨天快捷按钮选中态同步更新。
- 如果推断出博主，则对应博主按钮高亮。
- 标题预览自动刷新。

## 技术方案

### 1. manifest.json

增加读取当前标签页和 B 站投稿页内容所需权限：

```json
"permissions": ["clipboardWrite", "tabs"],
"host_permissions": ["https://member.bilibili.com/*"]
```

通过 `content_scripts` 将脚本限制在 B 站投稿页：

```json
"content_scripts": [
  {
    "matches": ["https://member.bilibili.com/platform/upload/*"],
    "js": ["content.js"],
    "run_at": "document_idle"
  }
]
```

### 2. content.js

新增 `content.js`，负责在当前页面中读取候选文本。

第一版读取范围：

- `input`
- `textarea`
- 常见可见文本节点
- 页面中可能承载文件名或标题的元素文本

返回内容：

```js
{
  success: true,
  candidates: ["2026-06-04 19-29-00 跟练七天看变化！.mp4"]
}
```

如果没找到候选文本：

```js
{
  success: false,
  message: "未找到候选文本"
}
```

### 3. popup.html

新增识别按钮，放在顶部作为高频入口：

```html
<button id="detectCurrentPage">识别当前投稿页</button>
```

识别摘要紧跟按钮展示，未识别前不占用额外高度。

### 4. popup.js

新增核心函数：

- `detectCurrentPageTitle()`
- `extractDateFromText(text)`
- `inferTrainerFromText(text)`
- `applyDetectedResult(result)`
- `showStatus(message)`

流程：

```text
点击“识别当前投稿页”
-> 获取当前 active tab
-> 检查 URL 是否为 B 站投稿页
-> 向投稿页 content.js 发送消息
-> 获取候选文本
-> 提取日期
-> 推断博主
-> 更新 selectedDate / selectedTrainer
-> renderTrainerButtons()
-> bindTrainerButtonEvents()
-> refreshControls()
-> 显示状态反馈
```

### 5. popup.css

新增识别按钮和状态提示样式：

- 识别按钮使用 secondary 样式，避免抢主操作“复制标题”的视觉权重。
- 状态提示使用小号文本，成功/失败用颜色区分。

## 非目标

第一版暂不实现：

- 自动监听 B 站投稿页变化。
- 自动写入 B 站标题输入框。
- 自动上传封面。
- 识别 `20260604`、`6月4日`、`06.04` 等扩展日期格式。
- 复杂博主判断或模糊匹配。

## 第二阶段：一键填写到 B 站投稿页

### 目标

在用户确认插件内标题预览无误后，点击一个明确的最终操作按钮：

```text
填写到 B 站
```

插件将当前预览标题和固定投稿配置写入当前 B 站投稿页。

第二阶段优先做“可控自动化”，不直接发布视频，不点击 B 站页面的投稿/发布按钮。

### 用户流程

推荐流程：

```text
1. 上传视频
2. 打开插件
3. 点击“识别当前投稿页”

7. 插件写入标题，并选择创作声明、分区、标签等配置
8. 插件显示填写结果摘要
```

### 第一版填写范围

建议分两步上线，降低风险。

#### 第二阶段 v1：只填写标题（已实现）

第一版只做：

- 将插件标题预览框中的当前文本写入 B 站标题输入框。
- 触发 B 站页面所需的 `input` / `change` / blur 事件，确保页面状态保存。
- 显示“已填写标题”或失败提示。

暂不自动选择：

- 创作声明
- 分区
- 标签
- 封面
- 定时发布
- 任何发布/提交按钮

理由：

- 标题输入框是最核心、最容易验证的自动化动作。
- B 站页面是动态表单，先验证写标题的选择器和事件触发方式是否稳定。
- 用户仍可通过“复制标题”兜底。

#### 第二阶段 v2：填写标题 + 投稿配置（已实现）

在标题写入稳定后，已增加：

- 创作声明：默认选择 `内容无需标注`
- 分区：默认选择 `健身`
- 标签：默认设置 `减肥`、`健身`、`训练`
- 标签清理：删除非默认干扰标签，再补齐默认标签

可以根据博主或训练类型进一步扩展标签：

| 条件 | 可选标签 |
| --- | --- |
| 祖嘉泽 | 有氧、跟练、燃脂 |
| 张峰 | 减脂、增肌、训练 |
| 陈玉轩 | 塑形、训练、健身 |
| 周五有氧类型 | 有氧、燃脂、训练 |

第二阶段 v2 仍然不点击最终发布按钮。

### 技术方案

#### 1. content.js 增加写入能力

当前 `content.js` 只读取页面候选文本。下一阶段增加消息类型：

```js
FILL_UPLOAD_FORM
```

请求示例：

```js
{
  type: 'FILL_UPLOAD_FORM',
  payload: {
    title: '「2026/06/05 周五｜晚间」祖嘉泽有氧健身 直播回放录屏完整版',
    declaration: '内容无需标注',
    category: '健身',
    tags: ['减肥', '健身', '训练']
  }
}
```

响应示例：

```js
{
  success: true,
  results: [
    { field: 'title', success: true, message: '已填写标题' },
    { field: 'declaration', success: true, message: '已选择创作声明：内容无需标注' },
    { field: 'category', success: true, message: '已选择分区：健身' },
    { field: 'tags', success: true, message: '已设置标签：减肥、健身、训练' }
  ]
}
```

部分失败时：

```js
{
  success: false,
  results: [
    { field: 'title', success: true, message: '已填写标题' },
    { field: 'category', success: false, message: '未找到分区控件，请手动选择' }
  ]
}
```

#### 2. 表单写入工具函数

需要在 `content.js` 中封装稳定的 DOM 操作：

- `setInputValue(element, value)`
- `dispatchInputEvents(element)`
- `clickByText(text, scope)`
- `findFieldByLabel(labelText)`
- `waitForElement(getter, timeout)`
- `fillTitle(title)`
- `selectDeclaration(value)`
- `selectCategory(value)`
- `setTags(tags)`

当前实现补充：

- 标题输入使用原生 `value` setter，并触发 `input`、`change`、`blur`。
- 创作声明使用 B 站 BCC select 组件适配，默认值为 `内容无需标注`。
- 分区通过字段 label 附近的选择器定位，并在当前下拉列表中选择 `健身`。
- 标签写入前会识别已选标签 chip，删除非默认标签，再输入缺失标签并回车创建。
- 字段失败时返回逐项结果，不阻塞其他字段继续写入。

写入 input/textarea 时，不能只改 `element.value`。需要触发事件：

```js
element.focus();
element.value = value;
element.dispatchEvent(new Event('input', { bubbles: true }));
element.dispatchEvent(new Event('change', { bubbles: true }));
element.blur();
```

如果 B 站使用 React/Vue 受控组件，可能还需要使用原生 setter：

```js
const setter = Object.getOwnPropertyDescriptor(
  HTMLInputElement.prototype,
  'value'
)?.set;
setter.call(element, value);
element.dispatchEvent(new Event('input', { bubbles: true }));
```

#### 3. popup.js 增加填写流程

新增核心函数：

- `fillCurrentUploadPage()`
- `buildFillPayload()`
- `requestFillUploadForm(tabId, payload)`
- `showFillResult(results)`

流程：

```text

```

### 选择器与稳定性策略

B 站投稿页可能频繁调整 DOM。不要只依赖单一 CSS class。

优先级策略：

1. 通过 label 文案定位字段，例如 `标题`、`创作声明`、`分区`、`标签`。
2. 在 label 附近查找 input、textarea、combobox、dropdown。
3. 通过 placeholder、maxlength、已知字段结构辅助判断。
4. 最后才使用具体 class 作为兜底。

标题输入框可参考特征：

- 位于 `标题` label 同一行或附近。
- 当前默认值形如 `2026-06-01 19-29-00 ...`。
- 右侧有 `28/80` 这类字数提示。
- 最大长度可能为 80。

创作声明字段可参考特征：

- label 文案为 `创作声明`。
- 下拉当前值可能为 `内容无需标注`。
- B 站 BCC select 可能把选项文本放在组件内部，选项不一定暴露为独立可见文本节点。

分区字段可参考特征：

- label 文案为 `分区`。
- 下拉当前值可能为 `健身`。
- 当前目标 `健身` 是分区下拉中的直接选项，不走二级类目。

标签字段可参考特征：

- label 文案为 `标签`。
- 可见已选标签 chip。
- 有提示 `按回车键Enter创建标签`。
- 自动填写时只保留默认标签 `减肥`、`健身`、`训练`，其他标签视为干扰标签。

### 安全策略

第一版必须遵守：

- 不点击 B 站最终发布按钮。
- 不删除用户已有标题，除非用户点击“填写到 B 站”。
- 写入前使用插件预览框当前文本，而不是重新生成标题，尊重用户手动修改。
- 写入后给出逐项结果，提示用户最终检查。
- 自动选择声明、分区、标签时，如果找不到目标选项，停止该字段并提示手动处理，不做模糊乱点。
- 不点击 B 站最终发布按钮，用户必须在 B 站页面最终确认后手动发布。

推荐增加确认策略：

```text
填写前不弹窗，保持高频效率。
填写后显示逐项结果，用户在 B 站页面最终确认。
```

如果后续自动化范围扩大到多个字段，可以考虑首次使用时提示：

```text
将填写标题、创作声明、分区和标签。不会点击发布。
```
